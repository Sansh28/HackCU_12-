# Contributing

## Getting Started

1. Read `README.md`
2. Read `docs/WORKFLOW.md`
3. Read `docs/DETAILED_WORKFLOW.md`
4. Start the backend, frontend, and extension as needed for your task

## Branching

- Create a focused branch for each change
- Keep documentation, structural refactors, and product features in separate branches when possible

## Pull Requests

- Summarize the problem and the solution
- Call out any env/config updates
- Include verification steps you ran
- Mention any remaining risks or follow-up work

## Code Changes

- Do not commit secrets
- Prefer non-destructive refactors
- Preserve git history for moved files
- Keep docs updated when structure or setup changes

## Verification Expectations

- Backend: run unit/integration tests when possible
- Frontend: run lint, TypeScript, and build checks when possible
- Extension: run typecheck and build when possible
